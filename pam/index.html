Capabilities

One coherent system. Forty domains.

PAM isn't a collection of tools stitched together. It's a single .NET solution where every module shares the same auth, the same audit trail, the same real-time bus, and the same back-office.

Multi-brand, multi-jurisdiction

One deployment runs all your brands. Per-brand skins, currencies, languages, and jurisdiction rules — without forking the codebase.

Real-time wallet

Optimistic concurrency on every wallet operation. SignalR push for balance changes — no polling, no stale balances on the player UI.

Banking + payments

PaymentIQ, Hexopay, SafeCharge. Deposit and withdrawal flows with bonus codes, fraud checks, manual approval queues, and idempotent callbacks.

Bonus engine

Welcome, deposit-match, free spins, cashback, tournament prize pools. Wagering requirements, expiry, and rollover all tracked.

KYC + responsible gaming

TransUnion, Experian, Acuris for identity. Source-of-wealth checks. Self-exclusion, loss limits, reality checks — all jurisdiction-aware.

Casino + sportsbook + bingo + lotto

25+ game providers integrated (NetEnt, Pragmatic, Evolution, NoLimitCity, …). Sports via SportTech / BetConstruct. Lotto via Lottoland.

RBAC permissions

Permission filters at controller and action level. Skin-scoped queries throughout. Audited writes everywhere.

Back-office

80+ controllers, 44+ view areas. Player search, KYC review, manual wallet adjust, withdrawal approval, bonus assignment — all from one operator portal.

AI module — built in

AI Chat and AI Studio with 60+ tools, provider-agnostic (Gemini / Claude / OpenAI). Operators get a coworker, not just a chatbot.

Architecture

Built for auditability and uptime.

PAM runs on familiar, auditable infrastructure for regulated markets. The event-driven design, caching, and messaging layers are cloud-compatible — and a migration path exists for operators who need cloud deployment.

  • ✓Player API — REST + real-time push for the player-facing site, with dual auth and full Swagger docs
  • ✓Back-office — operator portal with role-based access down to the individual action
  • ✓Business logic — 40+ feature modules, each behind a clear interface
  • ✓Background workers — behavior engine, scheduled jobs, and async event queues
  • ✓AI module — provider-agnostic, plugs in Gemini, Claude, or OpenAI without code changes
