Roles & Permissions (RBAC)

Permission Architecture

Three-tier hierarchy with inheritance and deny-list support
Permission Model
Role
  • Named role (e.g. "Support Manager", "Compliance Officer")
  • Inherits from parent role (ParentId)
  • Assigned to specific skins/brands
  • IsAdmin flag for superusers
Permissions[]
  • Admin Full system access
  • Controller Access to all actions in a controller
  • Action Granular single-action access
DeniedPermissions[]
  • Blacklist overrides grant
  • User-level overrides role
  • Prevents accidental privilege escalation

Permission Enforcement Flow

1
Staff User Navigates to a controller/action (e.g. Player/Account)
2
PermissionFilter Runs on every request. Checks [AllowAnonymous] first — skips if present.
3
Role Resolution Loads user's role + parent role chain. Merges Permissions[] with user-level overrides.
4
Deny Check Checks DeniedPermissions[] — deny always wins over grant.
5
Skin Check Verifies user has access to the current skin/brand being operated on.
6
Result Granted proceed to action — or 403 Forbidden + logged in audit trail

AI Module (PAM.AI)

Two AI-powered surfaces for back-office operators Powered by gemini-2.5-flash via Google GenAI. AI Chat is an inline panel on player pages for conversational, function-calling queries about a specific player. AI Studio is a full-page agentic system that generates a multi-step plan, executes steps against live data, and supports scheduled runs with Slack reporting.

AI Chat — Inline Player Panel

Function-calling flow with 60+ tools
1
Staff User Types a question in the AI chat panel on a player page. AIChatService builds an AIToolPermissionContext from the current user's permissions and skin.
2
PII Sanitization Player names, emails, phones, and addresses are replaced with [TOKEN_N] tokens before sending to Gemini. Tokens are restored in the final response.
3
Gemini 2.5 Flash GeminiService sends message + FunctionDeclarations for all permitted tools (filtered by AIToolRegistry). Gemini returns FunctionCall(s) if data is needed. Up to 5 function-calling rounds per message.
4
Tool Execution Each tool re-validates permissions and skin scope before executing. Result is returned as FunctionResponse → Gemini returns final text.
5
SignalR Push Response pushed to UI via IAIStudioNotifier / SignalR. Conversation persisted in AIChatMessages.

AI Studio — Agentic Sessions

Event-driven plan → execute loop; survives browser disconnects
1
Staff User Types a goal in AI Studio (/AIStudio). AIStudioHub.StartSession(goal) publishes ai_studio.session.requested to RabbitMQ and returns immediately.
2
PAM.Service.EventQueue AIStudioSessionListener consumes the event. AIStudioService.GeneratePlan(goal) calls Gemini → returns a JSON plan of typed steps (Query / Analyze / Write).
3
Plan Review Plan pushed via SignalR Redis backplane (PlanGenerated event). Single-step plans with no approval auto-execute; otherwise operator reviews and clicks Run.
4
Step Execution Each ExecuteStep call publishes ai_studio.step.requested. Listener calls GeminiService.Chat() with step prompt + tools → pushes StepCompleted with result HTML. Write steps always require operator approval before execution (ApproveStep publishes ai_studio.approval.requested).
5
Dynamic Replanning After the last Analyze step, TryReplan asks Gemini if additional steps are warranted. New steps are appended and pushed via StepsAppended. Fires at most once per turn.
6
Persistence Session always persisted to DB regardless of connection state. restoreLatest() rehydrates session state when operator returns.
